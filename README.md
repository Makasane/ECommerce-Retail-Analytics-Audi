##  3. Data Validation & Quality Assurance (The Retail Audit)

Before data modeling, a rigorous programmatic data validation audit was executed via Python. This stage mimics corporate risk mitigation by identifying system anomalies before they contaminate executive dashboards.

### Automated Integrity Checks Implemented:
* **Pricing Engine Logic Check:** Isolated and flagged data anomalies where current selling `price` mathematically exceeded the original baseline `listPrice` (System markup logic failure).
* **Cross-Field Behavioral Verification:** Screened for logical mismatches where a product record was flagged as a market driver (`isBestSeller = True`) but reported zero velocity in volume metrics (`boughtInLastMonth = 0`).
* **Completeness & Null Imputation:** Analyzed the dataset for structural gaps in product evaluation scores (`stars` and `reviews`), executing statistical drop/imputation protocols to preserve reporting weights.
* **Granular Cardinality Audit:** Evaluated unique constraint bounds across `category_id` allocations to eliminate duplication loops during database joins.

```python
# Conceptual layout of the programmatic audit script executed in Python
import pandas as pd

def run_retail_audit(file_path):
    df = pd.read_csv(file_path)
    
    # Check 1: Pricing Logic Anomaly Detection
    pricing_anomalies = df[df['price'] > df['listPrice']]
    
    # Check 2: Best Seller Velocity Integrity
    velocity_mismatches = df[(df['isBestSeller'] == True) & (df['boughtInLastMonth'] == 0)]
    
    print(f"[AUDIT COMPLETE] Isolated {len(pricing_anomalies)} pricing bugs and {len(velocity_mismatches)} velocity anomalies.")
    return df.drop_duplicates()
```

##  4. SQL Data Modeling & Semantic Layer Configuration

The validated dataset was migrated into a production table inside Google BigQuery. To fulfill the core "single source of truth" requirement sought by retail enterprise systems, complex Standard SQL queries were engineered to establish standardized dimensions, measures, and calculated fields.

### Engineered Commercial KPIs (Calculated Measures):
1. **Estimated Gross Monthly Revenue:** Generated by modeling total monthly sales metrics against absolute transaction pricing:
   $$\text{Estimated Monthly Revenue} = \text{price} \times \text{boughtInLastMonth}$$
2. **Markdown Discount Percentage:** Formulating commercial price elasticity margins across distinct inventory tiers:
   $$\text{Discount \%} = \frac{\text{listPrice} - \text{price}}{\text{listPrice}} \times 100$$
3. **Consumer Engagement Velocity:** A calculated operational weight score mapping structural brand health:
   $$\text{Engagement Velocity} = \text{stars} \times \text{reviews}$$

### High-Performance Production Query:
```sql
WITH AuditedRetailData AS (
  SELECT 
    CAST(category_id AS STRING) AS category_id,
    CAST(price AS NUMERIC) AS selling_price,
    CAST(listPrice AS NUMERIC) AS original_list_price,
    SAFE_CAST(boughtInLastMonth AS INT64) AS sales_volume,
    CASE WHEN isBestSeller = 'true' THEN 1 ELSE 0 END AS is_bestseller,
    SAFE_CAST(stars AS NUMERIC) AS product_rating,
    SAFE_CAST(reviews AS INT64) AS review_count
  FROM 
    `pepkor-speciality-simulation.retail_analytics.raw_ecommerce_data`
  WHERE 
    price <= listPrice -- Enforcement of programmatic pricing boundary logic
)
SELECT 
  category_id,
  SUM(sales_volume) AS total_units_sold,
  ROUND(SUM(selling_price * sales_volume), 2) AS estimated_monthly_revenue,
  ROUND(AVG((original_list_price - selling_price) / NULLIF(original_list_price, 0)) * 100, 2) AS avg_markdown_percentage,
  SUM(is_bestseller) AS total_bestseller_count,
  ROUND(AVG(product_rating), 2) AS weighted_customer_satisfaction
FROM 
  AuditedRetailData
GROUP BY 
  category_id
ORDER BY 
  estimated_monthly_revenue DESC;
```
